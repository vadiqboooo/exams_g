# Исправление проблемы с отображением групп для school_admin

## Проблема
Администраторам школ не отображаются группы из-за несоответствия названий школ:
- В базе данных группы используют русские названия: "Байкальская", "Лермонтова"
- Администраторы были созданы с английскими: "Baikalskaya", "Lermontova"

## Решение

### 1. Исправьте названия школ в базе данных

Запустите скрипт исправления:
```bash
cd backend
python fix_school_names.py
```

Вы должны увидеть:
```
Ispravlenie nazvanij shkol...
  [OK] Obnovlen admin_baikalskaya: school=Байкальская
  [OK] Obnovlen admin_lermontova: school=Лермонтова
[SUCCESS] Nazvanija shkol ispravleny!
```

### 2. Перезапустите backend сервер

Если backend уже запущен, остановите его (Ctrl+C) и запустите снова:
```bash
cd backend
uvicorn main:app --reload
```

### 3. Перелогиньтесь в приложении

**ВАЖНО**: Старый JWT токен содержит неправильное значение школы!

1. Откройте приложение в браузере
2. Нажмите кнопку "Выйти" в правом верхнем углу
3. Войдите заново:
   - Username: `admin_baikalskaya`
   - Password: `baikalskaya123`

### 4. Проверьте результат

После входа перейдите на вкладку **"Группы"**:
- **admin_baikalskaya** должен видеть ~26 групп школы "Байкальская"
- **admin_lermontova** должен видеть ~4 группы школы "Лермонтова"

## Проверка (опционально)

Для проверки фильтрации можно запустить тест:
```bash
cd backend
python test_school_filter.py
```

Это покажет количество групп для каждого администратора.

## Почему это произошло?

Изначально скрипт `create_school_admin.py` использовал английские названия школ, но в базе данных группы уже были созданы с русскими названиями. Теперь все исправлено и использует единый формат: **русские названия**.
